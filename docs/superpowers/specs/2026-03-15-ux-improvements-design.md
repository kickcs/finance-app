# Спецификация: Улучшение UX — обнаруживаемость, обратная связь, мобильный UX

**Дата:** 2026-03-15
**Статус:** Утверждён
**Область:** Frontend (Vue 3, FSD архитектура)

---

## Проблема

Приложение имеет множество полезных функций, но пользователи не знают об их существовании. Особенно скрыты: сплит расходов, сканирование чеков и настройка дашборда. Также не хватает тактильной обратной связи и мобильных жестов.

## Цели

1. Повысить обнаруживаемость скрытых функций через контекстные подсказки и визуальные маркеры
2. Улучшить обратную связь после действий пользователя (toast с undo, haptics)
3. Оптимизировать мобильный UX (жесты, умные дефолты в формах)

---

## 1. Система контекстных подсказок (Feature Hints)

### 1.1 Визуальный стиль — Tooltip-bubble

Подсказка в виде всплывающего пузыря с указательной стрелкой, привязанного к конкретному элементу интерфейса.

**Содержимое подсказки:**
- Заголовок с иконкой лампочки (напр. "Совет: Разделите расход")
- Описание в одну строку (напр. "Нажмите «Разделить», чтобы поделить сумму с друзьями")
- Две кнопки: "Попробовать →" (переход к фиче) и "Не показывать" (dismiss)

**Стилизация:**
- Фон: indigo (`bg-primary`) с белым текстом
- Скругление: 12px
- Стрелка-указатель (CSS triangle) направлена на целевой элемент
- Анимация появления: `fadeInUp` 200ms

**Технология:** Reka UI `Tooltip` или `Popover` с кастомной стилизацией.

### 1.2 Триггеры показа

| Фича | Триггер | Где показывается | Указывает на |
|------|---------|-----------------|-------------|
| Сплит расходов | Пользователь создал 3+ транзакций типа "расход" | Форма добавления транзакции | Кнопка/область «Разделить» |
| Сканирование чеков | Пользователь вручную ввёл 5+ транзакций | Дашборд или форма транзакции | Кнопка сканирования |
| Настройка дашборда | Пользователь зашёл на дашборд 7+ раз | Верх дашборда | Кнопка ⚙️ настроек |

### 1.3 Правила поведения

- Каждая подсказка показывается **максимум 1 раз** (dismiss навсегда)
- Не более **1 подсказки за сессию** (сессия = с момента открытия приложения)
- Состояние хранится в `localStorage`:
  - `hints_dismissed: { [hintId]: true }` — какие подсказки закрыты
  - `hints_shown_this_session: boolean` — показывалась ли подсказка в текущей сессии
  - `hints_counters: { expenses_count, transactions_count, dashboard_visits }` — счётчики триггеров
- "Попробовать →" — навигация к фиче + dismiss
- "Не показывать" — только dismiss
- Подсказка закрывается также по клику вне неё

### 1.4 Архитектура

**Новый feature-модуль:** `features/feature-hints/`

```
features/feature-hints/
├── model/
│   ├── types.ts          # HintConfig, HintId
│   ├── constants.ts      # Конфигурация всех хинтов (триггеры, тексты, маршруты)
│   └── useFeatureHints.ts  # Composable: состояние, проверка триггеров, dismiss
├── ui/
│   └── FeatureHintTooltip.vue  # Компонент подсказки (Reka UI Tooltip/Popover)
└── index.ts
```

**Composable `useFeatureHints()`:**
- `checkAndShowHint(hintId): boolean` — проверяет триггер + не dismissed + не показан в сессии
- `dismissHint(hintId): void` — сохраняет в localStorage
- `incrementCounter(counter): void` — увеличивает счётчик (вызывается из мест создания транзакций, навигации)
- `activeHint: Ref<HintConfig | null>` — текущая активная подсказка

**Интеграция счётчиков:**
- `useTransactions` → при успешной мутации `create` вызывает `incrementCounter('transactions_count')` и `incrementCounter('expenses_count')` для расходов
- Роутер `HomePage` → при монтировании вызывает `incrementCounter('dashboard_visits')`

## 2. Пульсирующие точки обнаружения (Discovery Dots)

### 2.1 Визуальный стиль

Маленькая индиго-точка (8px) с пульсирующей анимацией, расположенная в правом верхнем углу элемента.

**Стилизация:**
- Цвет: indigo (`bg-primary`)
- Размер: 8px (10px на крупных элементах вроде кнопки +)
- Анимация: CSS `pulse` — масштабирование 1→1.8 с fade out, бесконечный цикл, 2s
- Позиционирование: `absolute`, `top: -2px`, `right: -2px`

### 2.2 Где показываются

| Элемент | Когда исчезает | Связанная фича |
|---------|---------------|----------------|
| Кнопка "+" (BottomNav) | После первого открытия формы транзакции с переходом к сплиту | Сплит расходов |
| Кнопка ⚙️ настроек дашборда | После первого входа в настройки дашборда | Настройка дашборда |
| Кнопка сканирования (если видна) | После первого использования | Сканирование чеков |

### 2.3 Архитектура

**Компонент:** `shared/ui/discovery-dot/DiscoveryDot.vue`

```vue
<DiscoveryDot id="split-expense" />
```

Props:
- `id: string` — уникальный идентификатор точки
- `size?: 'sm' | 'md'` — 8px или 10px (по умолчанию 'sm')

Состояние dismissed хранится в том же `localStorage` ключе, что и hints: `discovery_dots_dismissed: { [dotId]: true }`.

Компонент рендерится только если `!dismissed[id]`. При dismiss вызывает `useFeatureHints().dismissDot(id)`.

## 3. Улучшенный toast с undo

### 3.1 Визуальный стиль

Тёмный toast, появляющийся снизу экрана.

**Содержимое:**
- Слева: анимированная зелёная галочка в круге (32px)
- Центр: сумма (жирная, белая) + категория и счёт (серый, мелкий)
- Справа: кнопка "Отменить" (индиго текст на полупрозрачном фоне)

**Стилизация:**
- Фон: `#1e293b` (slate-800)
- Скругление: 14px
- Анимация появления: `slideUp` 300ms
- Авто-скрытие: 5 секунд
- Swipe-to-dismiss: свайп вниз закрывает toast

**Анимация галочки:** SVG stroke с `stroke-dasharray` + `stroke-dashoffset` анимация (checkmark draw effect), задержка 200ms.

### 3.2 Логика undo

- При нажатии "Отменить":
  - Вызывается `deleteTransaction(id)` для только что созданной транзакции
  - Баланс счёта откатывается (через существующую логику delete)
  - Toast заменяется на короткий "Отменено" (1.5 сек)
  - Haptic: `light`

### 3.3 Архитектура

Расширение существующего `shared/lib/composables/useToast.ts`:

- Новый вариант toast: `variant: 'transaction-success'`
- Дополнительные поля: `amount`, `categoryName`, `accountName`, `transactionId`, `onUndo`
- Компонент `TransactionSuccessToast.vue` внутри toast-системы
- Swipe-to-dismiss через `useSwipe()` из `shared/lib/hooks/`

## 4. Расширенные haptic-паттерны

### 4.1 Новые точки haptic-обратной связи

| Действие | Тип haptic | Где вызывается |
|----------|-----------|---------------|
| Создание транзакции (успех) | `success` | `useTransactions` → onSuccess мутации create |
| Удаление транзакции | `warning` | `useTransactions` → onSuccess мутации delete |
| Undo в toast | `light` | TransactionSuccessToast → onUndo |
| Ошибка валидации формы | `error` | TransactionForm → onValidationError |
| Dismiss подсказки | `light` | FeatureHintTooltip → onDismiss |
| Pull-to-refresh (триггер) | `success` | PullToRefresh → onRefresh |
| Порог свайпа транзакции | `selection` | SwipeableItem → onThreshold |

### 4.2 Реализация

Все вызовы через существующий `useHaptics()` → `trigger(type)`. Никаких новых зависимостей не требуется.

## 5. Асимметричный свайп на транзакциях

### 5.1 Поведение

- **Свайп влево** → открывает красную зону "Удалить" (с иконкой 🗑️)
- **Свайп вправо** → открывает индиго зону "Изменить" (с иконкой ✏️)

### 5.2 Анимация и тактильность

- Пружинная анимация при отпускании (spring easing)
- Haptic `selection` при достижении порога свайпа
- Порог для срабатывания действия: 80px
- Full-swipe (>200px): автоматическое выполнение действия

### 5.3 Архитектура

Модификация существующего `shared/ui/SwipeableItem.vue`:

- Новый prop: `swipeMode: 'left' | 'right' | 'both'` (по умолчанию 'both')
- Prop `leftAction: { icon, label, color, handler }`
- Prop `rightAction: { icon, label, color, handler }`
- Обратная совместимость: если передан только `leftAction`, поведение как текущее

Компоненты `TransactionItem` и `VirtualGroupedTransactionList` обновляются для использования нового `swipeMode="both"`.

## 6. Умные дефолты в форме транзакции

### 6.1 Логика автозаполнения

При открытии формы добавления транзакции:

1. Берём последние 20 транзакций пользователя
2. Группируем по категории → считаем частоту использования каждой пары (категория, счёт)
3. Предзаполняем:
   - **Категория:** самая частая для выбранного типа (расход/доход)
   - **Счёт:** самый частый для выбранной категории
4. Если у пользователя < 5 транзакций — не предзаполняем (недостаточно данных)

### 6.2 UX-поведение

- Предзаполненные поля визуально отличаются (слегка приглушённый стиль или подпись "Частый выбор")
- Пользователь может изменить в любой момент
- Не предзаполняем сумму и дату (всегда сегодня)
- Quick Actions имеют приоритет — если транзакция создана из quick action, его значения не перезаписываются

### 6.3 Архитектура

**Новый composable:** `features/add-transaction/model/useSmartDefaults.ts`

```typescript
useSmartDefaults(transactions: Transaction[], type: 'expense' | 'income')
// Возвращает: { defaultCategory, defaultAccount }
```

- Работает на клиентской стороне, анализирует уже загруженные транзакции из кеша Vue Query
- Никаких новых API-запросов
- Интегрируется в `TransactionForm.vue` через `watch` на тип транзакции

## Общие технические решения

### localStorage-ключи

```
finance_app_hints_dismissed      — Record<string, boolean>
finance_app_hints_session_shown  — boolean (сбрасывается при перезагрузке)
finance_app_hints_counters       — Record<string, number>
finance_app_discovery_dots       — Record<string, boolean>
```

Все ключи с префиксом `finance_app_` для избежания коллизий.

### Зависимости

- **Reka UI:** Tooltip/Popover для хинтов (уже в проекте)
- **Reka UI Presence:** Для анимаций появления/исчезновения (уже в проекте)
- Никаких новых npm-зависимостей

### Компоненты Reka UI для использования

- `Tooltip` — для tooltip-bubble хинтов
- `Popover` — альтернатива, если нужен интерактивный контент внутри (кнопки dismiss/action)
- `Presence` — анимация монтирования/демонтирования подсказок и точек
- `Toast` — расширение существующей системы (уже кастомная)

### Порядок реализации

1. **Feature Hints** (composable + компонент + интеграция счётчиков)
2. **Discovery Dots** (компонент + интеграция в BottomNav/Dashboard)
3. **Toast с undo** (расширение toast-системы + TransactionSuccessToast)
4. **Haptic-паттерны** (точечные добавления trigger() вызовов)
5. **Асимметричный свайп** (модификация SwipeableItem)
6. **Умные дефолты** (composable + интеграция в TransactionForm)

### Что НЕ входит в скоуп

- Онбординг-тур для новых пользователей (отдельная задача)
- Баннер/хаб "Все возможности" (решили не делать)
- Кастомный numpad (отложено)
- Sticky-сумма в форме (отложено)
- Десктоп-специфичные улучшения (breadcrumbs и т.д.)
