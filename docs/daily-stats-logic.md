# Логика расчёта "Средний расход" и "Безопасный остаток"

## 1. Средний расход (/день)

### Формула

```
Средний расход = Сумма расходов за период / Количество дней в периоде
```

### Как считается количество дней (`daysInPeriod`)

```
daysInPeriod = ceil((endDate - startDate) / (1000 * 60 * 60 * 24))
// Минимум 1 день (защита от деления на 0)
```

Доступные периоды:

- **С начала недели** — с понедельника текущей недели до сегодня
- **С начала месяца** — с 1-го числа текущего месяца до сегодня (по умолчанию)
- **С начала года** — с 1 января до сегодня
- **Произвольный** — пользовательский диапазон дат

### Сумма расходов (`totalExpense`)

Берётся из API `/api/analytics/stats` с учётом выбранных фильтров (период, счета). Все валюты конвертируются в основную валюту пользователя через курс обмена.

### Пример расчёта

```
Период: "С начала года" (1 января — 13 марта = 72 дня)
Общие расходы за 72 дня: 34 006 248 UZS

Средний расход = 34 006 248 / 72 = 472 309 UZS/день
```

## 2. Безопасный остаток (/день)

### Формула

```
Безопасный остаток = Доступный баланс / Дней до конца месяца
```

### Как считается доступный баланс (`availableBalance`)

1. Берутся счета: если выбраны конкретные — только они, иначе — все счета типа `basic`
2. Для каждого счёта суммируются балансы по всем валютам, конвертированные в основную валюту
3. Для кредитных карт: `Math.max(0, balance)` — отрицательный баланс не учитывается

```
availableBalance = Σ (баланс каждого подходящего счёта, конвертированный в основную валюту)
```

### Как считаются оставшиеся дни (`daysRemainingInMonth`)

```
daysRemainingInMonth = Последний день месяца - Сегодняшнее число
```

Например, 13 марта: `31 - 13 = 18 дней`

Особый случай: если `daysRemainingInMonth <= 0` (последний день месяца), показывается "Последний день" и возвращается весь доступный баланс без деления.

### Пример расчёта

```
Дата: 13 марта 2026
Дней до конца месяца: 31 - 13 = 18

Общий баланс на "basic" счетах: 12 289 212 UZS

Безопасный остаток = 12 289 212 / 18 = 682 734 UZS/день
```

## 3. Статус-индикатор (цветная подсказка)

Сравнивается **безопасный остаток** с **средним расходом**:

| Условие                        | Статус             | Сообщение                              |
| ------------------------------- | ------------------ | -------------------------------------- |
| `safeDaily <= 0`                | danger (красный)   | "Внимание! Расходы превысили доходы"   |
| `safeDaily < avgDailyExpense`   | warning (жёлтый)   | "Осторожно! Расходы превышают норму"   |
| `safeDaily >= avgDailyExpense`  | good (зелёный)     | "Отличный результат! Вы в плюсе"      |

### Пример

```
Безопасный остаток: 682 734 UZS/день
Средний расход:     472 309 UZS/день

682 734 >= 472 309 → статус "good" → зелёный: "Отличный результат! Вы в плюсе"
```

Логика: если можешь тратить в день больше, чем в среднем тратишь — значит всё хорошо.

---

## 4. Детали поведения

- **Карточка "Безопасный остаток" скрывается**, если выбран прошлый период (`isPastPeriod`) — нет смысла показывать, сколько можно тратить задним числом
- **Отрицательный `safeDaily` отображается как 0** — `Math.max(0, safeDaily)` в шаблоне
- **`balanceLabel`** — если выбраны конкретные счета, показывается пометка "По выбранным счетам"
- Безопасный остаток считается **всегда до конца текущего месяца**, независимо от выбранного периода аналитики (даже если период "с начала года")
